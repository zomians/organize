# CONTEXT.md

zomians/organize の用語集。この案件固有の概念のみを正準定義する。一般的なプログラミング概念は載せない。

## Language

**organize**:
「情報を整理したい」と思った瞬間から終わりまでを一気通貫で支援する Claude Code plugin。本案件そのもの。

**伴走 (ナビ)**:
各 skill が「次に何をするか」を能動的に示し、利用者の迷いをゼロにする organize の中核価値。静的な足場（地図）を置くのではなく、整理という行為に動的に付き添う。
_Avoid_: オーケストレーション（中央集権の統括は採らない）

**pwd**:
現在地判定と現状把握の対話を兼ねる入口 skill。利用者が今どのフェーズにいるかを判定し、最初のフェーズを兼ねる。一問一答・推奨案付きで決定木を一枝ずつ潰す。
_Avoid_: grill（出自として影響を受けた語だが、organize の現役語彙としては使わない）、ヒアリング、インタビュー

**壁打ち**:
現状把握で行う、計画を鍛えるための対話。質問は情報を集めるためでなく、矛盾・曖昧さ・未検証の前提を突いて計画を鍛えるためにある。
_Avoid_: ヒアリング、インタビュー（集めるための質問ではない）

**Doc Catalog (Catalog)**:
plugin に同梱された doc の辞書。pwd が signal から逆引きする単一の真実（該当 doc が無ければ作成を提案、有ればその場で更新）。案件側の CLAUDE.md には置かない。
_Avoid_: テンプレート集、ドキュメント一覧

**signal**:
会話中に surface する、特定の doc が必要だと示す合図。pwd はこれを検出して Catalog を逆引きする（走査ではなく逆引き）。
_Avoid_: トリガー（skill 発火のトリガーとは別概念）

**出口**:
pwd の対話が一段落した後の分岐。doc に残す / issue 直行 / 両方 / 何も残さず / handoff の 5 通り。
_Avoid_: 完了、エンド
