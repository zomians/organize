# CONTEXT.md

zomians/organize の用語集。この案件固有の概念のみを正準定義する。一般的なプログラミング概念は載せない。

## Language

**organize**:
「情報を整理したい」と思った瞬間から終わりまでを一気通貫で支援する Claude Code plugin。本案件そのもの。

**伴走 (ナビ)**:
各 skill が「次に何をするか」を能動的に示し、利用者の迷いをゼロにする organize の中核価値。静的な足場（地図）を置くのではなく、整理という行為に動的に付き添う。
_Avoid_: オーケストレーション（中央集権の統括は採らない）

**spar**:
整理の入口で計画を鍛える壁打ち skill。現状把握の最初のフェーズを兼ねる。一問一答・推奨案付きで決定木を一枝ずつ潰し、矛盾・曖昧さ・未検証の前提を突く。
_Avoid_: grill（出自として影響を受けた語だが、organize の現役語彙としては使わない）、ヒアリング、インタビュー

**壁打ち**:
現状把握で行う、計画を鍛えるための対話。質問は情報を集めるためでなく、矛盾・曖昧さ・未検証の前提を突いて計画を鍛えるためにある。
_Avoid_: ヒアリング、インタビュー（集めるための質問ではない）

**patrol**:
整理の第二の入口 skill。GitHub リポを巡回し、organize の核心（言語化 / GitHub flow）を診断レンズに「言語化負債 + flow 負債」を活動量でスケールして測り、「考えた方が良いこと」を発掘して spar / アクションに渡す。spar が悩みを内から引き出すのに対し、patrol は外から論点を持ち込む（向きが逆）。結論は出さず、揮発リストを置いて人間が 1 件選ぶ。
_Avoid_: 督促チェッカー（doc 不在を機械的に flag しない＝ lazy 維持）、triage tool（結論を出す道具ではない）、巡回設定ファイル（config は持たない）

**負債（言語化負債 / flow 負債）**:
patrol が測る、リポの「複雑さ × 活動量」に対して言語化 / GitHub flow の規律が追いついていない差分。言語化負債＝目的・用語・構造・決定が未言語化（→ 各 doc）。flow 負債＝作業が issue → PR → merge の規律に乗っていない（main 直 commit・放置 PR 等）。doc 不在そのものは負債でない（活動でスケールするため若いリポは足切り）。
_Avoid_: 技術的負債（コード品質一般の話ではない）

**Doc Catalog (Catalog)**:
plugin に同梱された doc の辞書。spar が signal から逆引きする単一の真実（該当 doc が無ければ作成を提案、有ればその場で更新）。案件側の CLAUDE.md には置かない。
_Avoid_: テンプレート集、ドキュメント一覧

**signal**:
会話中に surface する、特定の doc が必要だと示す合図。spar はこれを検出して Catalog を逆引きする（走査ではなく逆引き）。
_Avoid_: トリガー（skill 発火のトリガーとは別概念）

**出口**:
spar の対話が一段落した後の分岐。残す（issue 起点で main 反映、md は issue に添える）/ 何も残さず終える / handoff して中断（揮発）の 3 通り。
_Avoid_: 完了、エンド

**自動実行（自動実行モード）**:
「残す」出口で内包 skill（create-issue → commit → create-pr）を、事前確認ゲートを畳んでフォアグラウンドで PR 作成＋ブラウザ表示＋diff レビュー（read-only）まで連続実行すること。終点は PR の成果物（PR ＋ diff レビュー結果。merge と findings に基づく修正は人間）。発火は spar の「PRまで自動実行」yes の連鎖に限る（素性トリガ・fail-closed）。
_Avoid_: 委譲（subagent に渡すことではない）、オーケストレーション（中央統括ではない）、丸投げ（成果物で検証できる実行フェーズに限る）
