# ADR 0001: 伴走は両端に集約し、実行フェーズに限り subagent 委譲を認める

**文脈**: organize の中核価値「伴走」は _Avoid_ にオーケストレーション（中央集権の統括）を置くが、subagent 委譲は放っておくと丸投げ／オーケストレーションに滑り、この原則と衝突しうる。

**決定**: subagent への委譲はレビュー可能な成果物が返る実行タスクに限り、人間は入口（spar＝入力の擦り合わせ）と出口（成果物レビュー＝出力の検証）の両端にだけ居る。成果物が返らない真の探索は委譲せず spar で人間がやる。中央のオーケストレータは置かない。

**理由**: 中間（subagent の自律実行）の不透明さと throughput を引き換えに、人間が両端に集約して居続けることで伴走を壊さない、という実トレードオフを取る。「伴走 ≠ オーケストレーション」と「委譲を認める」が両立する理由は文脈なしには驚くため pin する。
