# ADR-0001: spar 単一エントリから第二の入口 patrol を追加する

- ステータス: 採用
- 日付: 2026-06-18

## 文脈

ARCHITECTURE は当初「整理の入口に壁打ち skill（spar）を置く」＝単一エントリを構造の根拠にしていた。だが spar は「頭に悩み・課題がある」状態を前提とし（CONTEXT で「計画を鍛える壁打ち」と狭く定義）、「何を整理 / 作業するか自体を探す」フェーズ（悩みが未言語化、またはやることが外部に散らばっている状態）には適さない。

## 決定

spar と並ぶ第二の入口 skill `patrol` を追加し、入口を 2 つにする。patrol は GitHub 巡回で「考えた方が良いこと」を負債レンズで発掘し、spar（鍛える）または直接アクションへ放射状に渡す。

## 理由

「探す」は spar と情報の向きが逆（外 → 内）で独立した存在意義がある。spar に分岐を増やすと spar の輪郭がぼやけ、`_Avoid_: ヒアリング` の境界も崩れる。第二の skill として分離する方が、放射状・疎結合・各 skill が末尾で次を促す既存の座組みに素直に乗る。プラットフォーム化（plugin in plugin）は「層を増やすな／分類論が再燃する」に反するため採らない — 巡回先が増えても skill を足せばよい。

3 条件の確認: 後戻り困難（入口の数は organize の identity に直結し、利用者が依存する skill の撤回は難しい）／文脈なしには驚く（「spar は単一エントリ」と読んだ者が patrol を見ると驚く）／実トレードオフ（単一エントリの単純さ vs 二入口の網羅性、分類論再燃リスク）。
