# BRIEF.md

zomians/organize の目的・スコープ・成功基準。

## 目的

**「情報を整理したい」と思った瞬間から終わりまでを一気通貫で支援する Claude Code plugin。**

整理という行為に、最初から最後まで伴走する。各 skill が「次は何をするか」を能動的に示し、利用者が迷う箇所をなくす。

## スコープ

支援対象は、情報を整理する一連の流れ:

```
現状把握 (壁打ち / リサーチ)
   ↓
[ 出口の分岐 ]
 ├─ 残す (issue 起点・md は添える)
 ├─ 何も残さず終える
 └─ 引き継いで中断する (handoff)
   ↓
[ 残すと決めたものは ]
 リポジトリへ反映 (新規なら repo 化 / 既存なら該当リポへ)
   ↓
 issue → 作業ブランチ → PR → squash merge → branch 削除
```

含むもの:現状把握の壁打ち、必要時のリサーチ、ドキュメント化、リポ化、issue 起票、commit、PR 完了。

含まないもの:ローカルに残すだけと決めた情報の管理・検索・棚卸し（散らかりが痛くなってから足す。YAGNI）。

## 成功基準

- 「情報を整理したい」と思ってから迷う箇所がゼロになる（次に何をするかを常に skill が示す）
- 残すべき情報は GitHub に高クオリティな issue/PR 履歴つきで残り、そうでないものはローカルに残る
- skill の改善が plugin update だけで全ディレクトリに反映される
