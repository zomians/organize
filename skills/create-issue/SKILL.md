# create-issue

GitHub issue を同梱の `template.md` に沿って起票する。テンプレートはこのファイルに複製せず、`template.md` を単一の真実として毎回読む。

## 手順

1. **材料集め**: 会話・リポジトリ・関連 doc（BRIEF.md / ARCHITECTURE.md 等）から、テンプレートを埋めるのに必要な情報を集める。コードやリポジトリを見れば分かることは利用者に聞かない。
2. **下書き**: `template.md` の全セクションを埋めた本文を下書きする。
   - 下書きファイルは作らず、本文は会話で提示する
   - 該当しないセクションは `N/A`（一言理由つき）とし、セクション自体は削らない
   - 埋められない重要な空欄だけ、一問一答・推奨案付きで利用者に確認する。優先度・難易度は必ず推奨値と根拠を添える
   - 工数見積は確実性レベルを選び、バッファ込みの最終見積まで出す
3. **タイトル**: Conventional Commits の型に合わせる（例: `feat: ○○を追加する`）
4. **起票**: 下書きを利用者に提示して確認を得てから起票する。本文は stdin で渡す:
   `gh issue create --title "<title>" --body-file -`
5. **次へ**: 起票後、すぐ着手するか問う。着手するなら `feature/<#>-<summary>` ブランチを切って作業フェーズへ進む。

## 禁止

- issue 本文に AI 生成の旨を記載しない（`🤖 Generated with...` 等を付けない）
- 機密情報（トークン・内部 URL 等）を本文に含めない
- テンプレートの構造を勝手に変えない。不足セクションが必要なら利用者に確認してから足す
